Eigene Komponenten für Vaadin mit GWT und HTML5
==============================================================

Abstract
--------

In diesem Beispiel wird eine eigene Vaadin Komponente mit Hilfe des Google Web Toolkits entwickelt. Gezeigt wird die Herangehensweise bei der Entwicklung eigener Vaadin-Komponenten, die Verwendung von GWT Client Bundles und des HTML5-Features Local Browser Storage.

- - - - - - - - -

* __Artikel__: ["Eigene Komponenten für Vaadin mit GWT und HTML5"](http://www.oio.de/public/articles.htm)

* __Autor__: Roland Krüger

* __Veröffentlicht__: xx. Februar 2013

* __Sprache__: Deutsch

* __Vaadin Version__: ab 7.0

Bauen und ausführen
-----

Um das Beispiel auszuführen, wird [Apache Maven](http://maven.apache.org) benötigt. In dem Projektverzeichnis muss einfach das Kommando

$ mvn package jetty:run

ausgeführt werden. Das Maven-Kommando 'mvn' muss dafür im PATH liegen. Wenn das Projekt gebaut wurde und der Jetty-Container gestartet ist, öffnet man die Beispielanwendung in einem Browser über die URL http://localhost:8080/ResettableTextArea
